# Policy Visualizer Release Notes — 2.0.3

## Release

- Version: `2.0.3`
- Release date: `2026-03-04`
- Release type: `patch`
- Commit/tag: `v2.0.3`

## Summary

Patch release delivering a typed edge label contract for the Flow IR, API warning
propagation to the frontend, and upload limit alignment between FastAPI and nginx.
No breaking changes.

## Highlights

- Flow IR edges now carry a typed label enum (`YES`/`NO`/`match`/`default`/`auth-success`/`auth-fail`)
- API soft-fail warnings are now surfaced as a dismissable banner in the frontend
- nginx `client_max_body_size` aligned with FastAPI's 10 MB `MAX_UPLOAD_BYTES` limit

## Scope for This Release

- Included:
  - `src/flow_ir.py`: typed edge label values enforced at compile time
  - `api/schemas.py`: `EdgeLabel` enum added to wire format contract
  - `frontend/src/components/FlowDiagram.tsx`: edge rendering updated for typed labels
  - `frontend/src/App.tsx`: dismissable warning banner for `flow.warnings`
  - `frontend/src/api.ts`: `warnings: string[]` added to `FlowIR` interface
  - `frontend/nginx.conf`: `client_max_body_size` corrected from 50 M → 10 M
  - `docs/release-map-2.0.md`: post-GA backlog triage
  - `README.md`: upload limit and soft-fail warning behaviour documented
- Not included (still deferred):
  - UX-230, UX-231, UX-233: session drag persistence, auto-align, multi-service views
  - COL-240/241: vertical chain collapse/expand
  - ANN-223: WYSIWYG annotation editor
  - ISE integration (3.0 track)

## Breaking Changes

- None

## Features

### Diagram UX

- Warning banner: when the compiler emits soft-fail warnings (e.g. unresolved object
  references), an amber dismissable alert is now displayed above the diagram. Resets
  on each new file load.

### Policy/Compiler Pipeline

- **Typed edge label contract**: `src/flow_ir.py` now enforces a closed set of edge
  label values (`YES`, `NO`, `match`, `default`, `auth-success`, `auth-fail`).
  `api/schemas.py` exposes this as an `EdgeLabel` enum in the wire format.

### API

- None (schemas updated to reflect typed labels; no endpoint changes)

## Fixes

- **FIX**: `frontend/src/api.ts` — `FlowIR` interface was missing `warnings: string[]`
  field; API responses containing warnings were silently dropped.
- **FIX**: `frontend/nginx.conf` — `client_max_body_size` was 50 M but FastAPI enforces
  10 M (`MAX_UPLOAD_BYTES`); the mismatch caused nginx to accept uploads that the API
  would then reject with 413. Now aligned at 10 M.
- **FIX**: `README.md` — documented the 10 MB upload limit and soft-fail warning behaviour.
