# Policy Visualizer Release Notes — 4.0.0-beta.2

## Release

- Version: `4.0.0-beta.2`
- Release date: `2026-05-01`
- Release type: `beta`
- Commit/tag: `v4.0.0-beta.2`

## Summary

Hardening and schema-normalization pass on the Policy Builder introduced in
`4.0.0-beta.1`. Fixes blank role labels in the builder preview, adds a
ClearPass API schema normalization layer so that live-API payloads map cleanly
to builder canvas state, hides the ISE connection option from the UI until the
ISE condition builder is ready, and adds TACACS-aware end-node labelling plus
an export date stamp across all export formats.

The existing Visualize tab (XML upload → diagram) is unchanged.

## Highlights

- ClearPass API schema normalization layer: live REST payloads are reliably
  mapped to canonical builder canvas state regardless of ClearPass API version
- TACACS-aware end-node labels: diagrams now display "Access: ALLOW (TACACS)"
  / "Access: DENY (TACACS)" for TACACS service flows
- Export date stamp: PNG, SVG, and PDF exports include a generation timestamp
  in the filename
- Blank Set Role labels resolved in builder preview

## Scope for This Release

- Included:
  - ClearPass API schema normalization layer (`src/clearpass_client.py`)
  - TACACS-aware end-node sub_label in flow compiler and renderer
  - Export filename date stamp (frontend ExportPanel)
  - Blank Set Role label fix (flat `role_name` normalization in builder preview)
  - ISE option hidden from ConnectionPanel UI (backend client intact)
  - 53 additional tests (344 total, up from 291 in beta.1)

- Not included (deferred to 4.0.0 GA):
  - Write-back / push policy to appliance (Phase 2)
  - ISE condition builder (attribute dictionary fetch not yet implemented for ISE)
  - Nested condition groups (ConditionBuilder is flat AND/OR only)
  - TACACS-specific enforcement handling in the builder

## Breaking Changes

- None. All existing API endpoints and wire formats are unchanged.

## Features

### Policy Builder — Backend

- `src/clearpass_client.py`: Added schema normalization layer that maps
  ClearPass REST API response shapes to the canonical element format consumed
  by the builder canvas. Handles API version differences in field naming
  (e.g. `role_name` vs nested `role` object).

### Diagram / Flow Compiler

- TACACS-aware end-node labels: `flow_ir.py` and `ise_flow_ir.py` now emit
  a `(TACACS)` qualifier in the `sub_label` field of terminal end-nodes when
  the service type is TACACS, giving operators a clear visual distinction
  between RADIUS and TACACS flows.

### Export

- Export date stamp: the ExportPanel appends a `YYYYMMDD` date string to the
  default filename for PNG, SVG, and PDF exports
  (e.g. `MyService-20260501.png`).

## Fixes

- **Blank Set Role labels in builder preview**: Role Mapping rule rows showed
  empty labels when the ClearPass API returned a flat `role_name` string
  rather than a nested role object. Normalized at the schema layer so labels
  render correctly in the preview modal.
- **ISE option hidden from ConnectionPanel UI**: The ISE platform toggle is
  removed from the builder UI in this release to prevent confusion while the
  ISE condition builder is incomplete. The backend ISE client and proxy
  endpoints are fully intact.

## Tests and Validation

- Test command: `.venv/bin/pytest tests/`
- Result: **344 passed** (up from 291 in beta.1)
- New coverage:
  - Additional tests for ClearPass schema normalization edge cases
  - TACACS end-node label assertions in flow IR tests
  - Export filename format assertions

## Known Issues

- ISE condition builder not yet implemented (attribute dictionary fetch not
  wired for ISE; ConditionBuilder is ClearPass-only in this release)
- ConditionBuilder supports flat AND/OR only; nested condition groups not
  supported
- SSL verification warning UX: disabling SSL verify has no browser-visible
  warning (relies on user awareness)
