# Policy Visualizer Release Notes — 4.0.0-beta.1

## Release

- Version: `4.0.0-beta.1`
- Release date: `2026-03-29`
- Release type: `beta`
- Commit/tag: *(pre-release; no tag)*

## Summary

Introduces the **Policy Builder** — a major new mode that connects directly to a
live Aruba ClearPass or Cisco ISE appliance, browses its policy elements, and lets
you construct a new policy on a visual canvas and preview the resulting flow diagram.
This is Phase 1 (read + preview); write-back to the appliance is planned for 4.0.0 GA.

The existing Visualize tab (XML upload → diagram) is unchanged.

## Why 4.0 (Major Bump)

This release qualifies as a major version bump per the SemVer rules in CLAUDE.md
("major architecture overhaul"):

- Adds a fundamentally new paradigm: **create** policy (not just visualize XML exports)
- Introduces **live network connectivity** — the API now proxies authenticated requests
  to ClearPass/ISE REST endpoints at runtime
- New credential-bearing request model: credentials are sent per-request in the JSON
  body (stateless; never stored server-side)
- New endpoint `POST /api/flow/from-ir` has no XML-based equivalent

Existing API contracts (`POST /api/flow`, `POST /api/services`, `GET /api/health`)
are fully preserved.

## Highlights

- Connect to a live ClearPass or ISE appliance from the browser (credentials stay
  client-side; proxied per-request, never stored)
- Browse all policy elements (services, roles, auth methods/sources, enforcement
  profiles/policies) in a collapsible sidebar
- Drag elements onto a 4-node canvas (ServiceMatch → Auth → RoleMapping → Enforcement)
- Create a new service from scratch via the 3-step NewServiceWizard
- Load an existing policy as a template (TemplatePickerModal fetches full rule detail)
- Build conditions with ConditionBuilder (flat AND/OR, 13 operators, attribute
  dictionary fetched live from ClearPass)
- Edit rules inline with RuleEditor (add/remove/reorder, on_match toggle, default row)
- Preview the resulting flow diagram via `POST /api/flow/from-ir` (same FlowIR
  wire format as the XML pipeline)
- Export canvas state as JSON

## Scope for This Release

- Included:
  - ClearPass live API client (OAuth2, pagination, soft-fail)
  - ISE live API client (HTTP Basic, ISE 3.1+, soft-fail)
  - Builder proxy endpoints (7 new routes under `/api/builder/`)
  - `POST /api/flow/from-ir` — canvas-state → FlowIR compilation
  - Full builder frontend (14 components + API layer)
  - Visualize / Build tab bar in App.tsx
  - 46 new backend tests (291 total)

- Not included (deferred to 4.0.0 GA):
  - Write-back / push policy to appliance (Phase 2)
  - ISE condition builder (attribute dictionary fetch not yet implemented for ISE)
  - Nested condition groups (ConditionBuilder is flat AND/OR only)
  - TACACS-specific enforcement handling in the builder

## Breaking Changes

## Features

### Policy Builder — Backend

- `src/clearpass_client.py`: OAuth2 client-credentials flow; paginated HAL fetch;
  bulk `get_all_elements()` with per-resource soft-fail
- `src/ise_client.py`: HTTP Basic auth; ISE `{"version":…,"response":[…]}` envelope
  unwrapping; bulk `get_all_elements()` with soft-fail
- `api/schemas_builder.py`: Pydantic models for credentials, element lists, canvas
  state (BuilderCondition, BuilderRule, CanvasState hierarchy)
- `api/routes/builder.py`: 7 proxy endpoints — ClearPass connect/elements/attributes/
  policy-detail/service-detail; ISE connect/elements. 401/502 error mapping.
- `api/routes/flow.py`: `POST /api/flow/from-ir` — converts BuilderFromIRRequest
  to PolicyIR dataclasses (synthetic IDs via name-slug + SHA256) then calls the
  existing `compile_service()` function

### Policy Builder — Frontend

- `frontend/src/components/builder/BuilderView.tsx` — 3-column layout; drag-and-drop
  handlers; template converters from ClearPass REST API responses
- `frontend/src/components/builder/BuilderCanvas.tsx` — React Flow canvas with 4
  fixed nodes and pre-wired edges; toolbar (New Service / Preview / Export JSON)
- `frontend/src/components/builder/ConnectionPanel.tsx` — platform toggle
  (ClearPass / ISE), credential form, connect/disconnect, version display
- `frontend/src/components/builder/ElementsLibrary.tsx` — draggable element browser,
  collapsible sections, context-aware drag permissions
- `frontend/src/components/builder/BuilderSidePanel.tsx` — context inspector for
  each canvas node (service, auth, role mapping, enforcement)
- `frontend/src/components/builder/NewServiceWizard.tsx` — 3-step modal: name/type,
  match conditions, auth methods + sources
- `frontend/src/components/builder/ConditionBuilder.tsx` — flat AND/OR predicate
  list; namespace/attribute dropdowns; 13 operators
- `frontend/src/components/builder/RuleEditor.tsx` — numbered rule list; per-rule
  condition + action + on_match; reorder; default row
- `frontend/src/components/builder/TemplatePickerModal.tsx` — fetches policy/service
  detail from ClearPass; "Use Template" or "Blank" choice
- `frontend/src/components/builder/PreviewModal.tsx` — calls `/api/flow/from-ir`,
  renders FlowIRSchema in modal
- 4 canvas node-type components: ServiceMatchNode, AuthNode, RoleMappingNode,
  EnforcementPolicyNode
- `frontend/src/api/builderApi.ts` — typed fetch wrappers for all builder endpoints
  + CanvasState TypeScript types
- `frontend/src/App.tsx` — Visualize / Build tab bar; version watermark
