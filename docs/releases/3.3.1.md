# Policy Visualizer Release Notes — 3.3.1

## Release

- Version: `3.3.1`
- Release date: `2026-03-12`
- Release type: `patch`
- Commit/tag: `v3.3.1`

## Summary

Patch release improving the service-picker panel in the left sidebar. Adds a
live name-filter input with negation support, replaces the sort dropdown with
a compact icon button, and registers the service-match decision node in
`policy_details` so it is inspectable via `PolicyDetailsPanel`.

## Highlights

- Service list filters dynamically as you type; prefix `!` to exclude matches
- Sort A→Z / Z→A now a single icon button (cycles on click) instead of a dropdown
- Service match decision node (`{id}__match`) now has a `trace_rule_id` and
  returns full condition detail in the `PolicyDetailsPanel`

## Scope for This Release

- Included:
  - `frontend/src/components/UploadPanel.tsx` — name filter, negation hint, icon sort button
  - `src/flow_ir.py` — `trace_rule_id` on service match node
  - `src/policy_details.py` — service match entry added to rule index
- Not included:
  - Any backend API changes
  - ISE pipeline changes

## Breaking Changes

- None

## Features

### Diagram UX

- **Service name filter**: live substring filter on the service list; type `!foo`
  to exclude services whose name contains "foo". A contextual hint below the
  input explains negation syntax.
- **Sort icon button**: replaces the A→Z / Z→A dropdown. Clicking cycles through
  unsorted → ascending → descending; button highlights in indigo when a sort is
  active.

### Policy/Compiler Pipeline

- Service match decision node now carries a `trace_rule_id` (`{service_id}__match`),
  enabling `PolicyDetailsPanel` to display the full match condition when the node
  is selected.
- `build_clearpass_details` registers a `_rule_detail` entry for the service
  match node, including rendered condition text and "Proceed to authentication"
  action description.
