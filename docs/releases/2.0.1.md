# Policy Visualizer Release Notes — 2.0.1

## Release

- Version: `2.0.1`
- Release date: `2026-02-26`
- Release type: `patch`
- Commit/tag: `v2.0.1`

## Summary

Patch release addressing field-reported defects and real-world usability improvements
surfaced by first community contributor TiemcoM (PR #2 / Issue #1). Adds 8 ClearPass
operator aliases that were causing 422 errors on production exports, fixes a bug where
`ir.warnings` was never populated in API responses, and softens unresolved-reference
handling so partial XML exports (those missing built-in profiles/roles) produce a diagram
instead of an error.

## Highlights

- `BEGINS_WITH` and 7 other ClearPass operator aliases now supported in the normalizer
- Unresolved XML references (built-in profiles/roles not in export) produce placeholder
  nodes instead of a hard 422 error
- `warnings` array in API response now correctly carries unresolved-reference messages
- Service sidebar persists after selection; active service highlighted; filename displayed

## Scope for This Release

- Included:
  - Normalizer operator aliases: `BEGINS_WITH`, `MATCHES_REGEX`, `NOT_MATCHES_REGEX`,
    `BELONGS_TO`, `LESS_THAN_OR_EQUALS`, `GREATER_THAN_OR_EQUALS`, `EQUALS_IGNORE_CASE`
  - Soft-fail placeholder creation for unresolved roles, profiles, auth methods/sources
  - `ir.warnings` wired up to API response
  - Frontend sidebar and filename UX improvements
  - 7 new normalizer tests (78 total)
- Not included (still deferred):
  - UX-230, UX-231, UX-233: session drag persistence, auto-align, multi-service views
  - COL-240/241: vertical chain collapse/expand
  - ANN-223: WYSIWYG annotation editor
  - ISE integration (3.0 track)

## Breaking Changes

## Features

### Export
- No changes.

### Diagram UX
- Service list in the sidebar now remains visible after a service is selected
- Currently selected service button is highlighted (indigo border/background)
- Uploaded filename shown below the drop zone in a green pill

### Policy/Compiler Pipeline
- Added 8 new ClearPass operator alias mappings (see Fixes)
- Unresolved references now produce placeholder objects with a warning rather than a
  hard `ValueError`. Placeholder profiles have `profile_type="builtin"` and are
  treated as ALLOW (non-deny) in the flow graph — correct fallback behavior
- `PolicyIR.warnings` field now populated when placeholders are created

### API
- `POST /api/flow` response now includes a populated `warnings` array when the XML
  references built-in objects not present in the export

## Fixes

- **FIX**: `BEGINS_WITH` operator raised `ValueError` (unknown operator → 422). Now maps
  to `starts_with`, identical to `STARTS_WITH` semantics.
- **FIX**: 7 additional operator aliases were not mapped: `MATCHES_REGEX`, `NOT_MATCHES_REGEX`,
  `BELONGS_TO`, `LESS_THAN_OR_EQUALS`, `GREATER_THAN_OR_EQUALS`, `EQUALS_IGNORE_CASE`.
  All now resolve without error.
- **FIX**: `ir.warnings` was added to `PolicyIR` and `FlowIRSchema` in the PR but never
  populated — warning messages were only sent to the Python logger. Now each unresolved
  reference appends a message to `ir.warnings`, which is forwarded in the API response.
- **FIX**: Hard 422 errors for partial XML exports (referencing built-in profiles/roles not
  included in the export file). System now creates placeholder entries and continues.
