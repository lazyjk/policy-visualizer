## What's New in 3.2.0

### Compiler Pipeline — RADIUS Proxy Service Type

ClearPass exports can contain `RadiusProxyService` elements that represent
proxy-forwarded RADIUS authentication flows. These differ structurally from
standard RADIUS/TACACS services: they have no authentication methods, auth
sources, or role-mapping policy — only a match condition and an enforcement
policy list.

- **`src/parser.py`**: new `RadiusProxyService` extraction loop; sets
  `serviceType: "RADIUS_PROXY"` and captures `ProxyNameList` alongside
  `EnfPolicyNameList`. Auth/role-mapping fields are always empty for proxy
  services.
- **`src/flow_ir.py`**: `compile_service()` branches on `service_type`;
  RADIUS_PROXY services wire the service-match YES edge directly to the
  enforcement chain, omitting the authentication process node, auth-fail
  terminal, and role-mapping decision chain entirely.
- **`tests/fixtures/radius-proxy.xml`**: new ClearPass XML fixture (EDUROAM
  RADIUS PROXY) with a three-predicate AND match condition including a
  `NOT_BELONGS_TO` operator, `ProxyNameList`, and `EnfPolicyNameList`.
- **Tests**: 6 new tests across `test_parser.py` (4) and
  `test_api_routes.py` (2).
