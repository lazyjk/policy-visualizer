# Policy Visualizer Release Notes — 3.0.0

## Release

- Version: `3.0.0`
- Release date: `2026-03-04`
- Release type: `ga` (major)
- Commit/tag: `v3.0.0`

## Summary

3.0 GA adds full Cisco ISE support alongside the existing Aruba ClearPass pipeline,
ships the WYSIWYG annotation editor (ANN-223), and includes a Draw.io export (BETA).
Format is auto-detected on upload — no user selection required. This release closes
the 3.0 train.

## Highlights

- Cisco ISE XML exports now supported (RADIUS and TACACS policy sets) with
  automatic format detection
- WYSIWYG annotation editor — bold, italic, bullet lists, font controls,
  image embed, border styling (ANN-223 closed)
- Draw.io export (BETA) — one-click mxGraphModel XML for diagrams.net
- Grid-include toggle for PNG / SVG / PDF exports
- Dismissable multi-select hint tooltip
- 185 tests passing (74 new ISE tests)

## Breaking Changes

- **Service selection by ID (not name)**: `/api/flow` requires `service_id`
  (from the `id` field of `/api/services`). Previously `service_name` was used.
  API clients must be updated.
- **FlowEdge schema**: `reason` field added (empty string for ClearPass edges).
  Strict deserializers must handle the new field.
- **FlowEdgeLabel union expanded**: `"CONTINUE"` is now a valid edge label value.

## Features

### Cisco ISE Pipeline (new)

- **Parser**: Extracts policy sets (RADIUS + TACACS), library conditions, and profiles
  (Standard / TACACS / Commandset) from ISE XML exports.
- **Normalizer**: Maps 19 ISE camelCase operators to the canonical `Op` enum. Resolves
  REFERENCE condition nodes against the library with soft-fail warnings.
- **Policy IR**: `ISEPolicyIR` with `ISEPolicySet`, `ISEAuthenRule`, `ISEAuthorRule`,
  `ISEProfile`. DISABLED rules filtered; DISABLED policy sets retained for service picker.
- **Flow IR compiler**: Authentication rules render as process nodes with PASS / FAIL /
  CONTINUE edges. Authorization rules render as decision chains with profile-application
  action nodes.

### WYSIWYG Annotation Editor (ANN-223)

- Replaced plain textarea with Tiptap editor in AnnotationNode.
- Toolbar: bold, italic, bullet list, font family (5 options), font size (8 sizes),
  image upload, border style (dashed / solid / dotted / none), border color.
- Double-click to enter edit mode; blur or Escape to exit.
- NodeResizer active when annotation is selected.

### Export

- **Draw.io export (BETA)**: Generates mxGraphModel XML. Open in diagrams.net
  (File → Import from → Device) for further editing. Complex diagrams may need
  layout adjustments after import.
- **Grid toggle**: Checkbox in export panel controls whether the background grid
  is included in PNG / SVG / PDF exports.

### Diagram UX

- **Multi-select hint**: Dismissable tooltip at bottom-center ("Shift+click or drag...").
  Dismissal state persisted in localStorage.
- **Minimap removed**: Decluttered canvas.
- **Snap-to-grid removed**: Removed from annotation panel.
- Service picker shows RADIUS / TACACS badge next to each service name.

### API

- `_detect_format(data)` scans first 2 KB of XML for ISE vs ClearPass signatures;
  routes to the appropriate pipeline automatically.
- Both ClearPass and ISE now select services by `id`.

### Operator Enum Expansion

Six new canonical operators added to `src/normalizer.py` for ISE condition support:
`ip_equals`, `ip_not_equals`, `ip_greater_than`, `ip_less_than`, `not_in`.
