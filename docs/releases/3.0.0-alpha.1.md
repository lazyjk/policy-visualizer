## Summary

First alpha of the 3.0 train. Adds full Cisco ISE policy export support as a parallel
compiler pipeline alongside the existing Aruba ClearPass pipeline. Format is
auto-detected on upload ‚Äî no user selection required. Includes a breaking API change:
service selection is now by `id` rather than `name`.

## Highlights

- Cisco ISE XML exports now supported (RADIUS and TACACS policy sets)
- Auto-format detection in the API routes ‚Äî ClearPass and ISE handled transparently
- ISE pipeline reuses all existing IR primitives (BooleanExpr, FlowNode, FlowEdge, FlowIR)
- Service picker shows RADIUS / TACACS type badges
- 74 new ISE-specific tests; 185 tests total, all passing

## Breaking Changes

- **Service selection by ID (not name)**: `/api/flow` now requires `service_id` (from
  the `id` field of `/api/services` response). Previously `service_name` was used.
  API clients must be updated.
- **FlowEdge schema change**: `reason` field added (empty string for ClearPass edges).
  Clients that deserialize FlowEdge strictly must handle the new field.
- **FlowEdgeLabel union expanded**: `"CONTINUE"` is now a valid edge label value.

## Features

### Cisco ISE Pipeline (new)

- **Parser**: Extracts policy sets (RADIUS + TACACS), library conditions, and profiles
  (Standard / TACACS / Commandset) from ISE XML exports.
- **Normalizer**: Maps 19 ISE camelCase operators to the canonical `Op` enum. Resolves
  REFERENCE condition nodes against the library with soft-fail warnings. Reuses
  `BooleanExpr`/`Predicate`/`And`/`Or`/`Not` from the existing normalizer.
- **Policy IR**: Builds typed `ISEPolicyIR` with `ISEPolicySet`, `ISEAuthenRule`,
  `ISEAuthorRule`, `ISEProfile` dataclasses. DISABLED rules are filtered; DISABLED
  policy sets are retained for the service picker.
- **Flow IR compiler**: Compiles ISE policy sets into FlowIR. Authentication rules
  render as process nodes with PASS / FAIL / CONTINUE edges (CONTINUE triggered on
  `usernotfoundaction=CONTINUE`). Authorization rules render as decision chains with
  profile-application action nodes.

### Operator Enum Expansion

Six new canonical operators added to `src/normalizer.py` for ISE condition support:
`ip_equals`, `ip_not_equals`, `ip_greater_than`, `ip_less_than`, `not_in`.

### API

- `_detect_format(data)` scans the first 2 KB of uploaded XML for ISE vs ClearPass
  signatures and routes to the appropriate pipeline automatically.
- Service selection changed from name-based to ID-based across both pipelines.

### Frontend

- Service picker list now shows a RADIUS / TACACS badge next to each service name.
- Service selection passes `svc.id` internally (no visible change to users).
