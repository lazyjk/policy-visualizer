- Version: `3.1.0`

## Summary

Minor release adding evaluate-all rule combining support to the ClearPass
compiler pipeline and session persistence with annotation copy/paste to the
frontend. This document also summarises the two patch releases (3.0.1 and
3.0.2) that shipped in the same cycle.

---

## What's New in 3.1.0

### Compiler Pipeline — Evaluate-All Rule Combining

ClearPass enforcement policies and role-mapping policies can be configured with
a `ruleCombiningAlgorithm` of `evaluate-all`, meaning every rule is evaluated
and all matching actions fire simultaneously (rather than stopping at the first
match). The compiler pipeline now handles this mode end-to-end.

- **`src/parser.py`**: extracts `ruleCombiningAlgorithm` from
  `EnforcementPolicy/<Policy>` elements.
- **`src/policy_ir.py`**: new `rule_combine_algo` field on `EnforcementPolicy`
  dataclass (`"evaluate-all"` | `"first-applicable"`, defaults to
  `"first-applicable"`).
- **`src/flow_ir.py`**: in evaluate-all chains, non-last action nodes emit a
  `CONTINUE` edge to the next decision node, reusing the existing dashed-gray
  frontend rendering for continue paths.
- **`tests/fixtures/EvaluateAll.xml`**: new fixture covering evaluate-all
  enforcement and role-mapping policies.
- **Tests**: 16 new tests across `test_parser.py`, `test_policy_ir.py`, and
  `test_flow_ir.py`.

### Frontend — Session Persistence

Diagram state (loaded XML, selected service, node positions, annotation
content) is now persisted to `localStorage` and restored on page reload. Users
no longer lose their work after an accidental refresh.

### Frontend — Annotation Copy/Paste

Annotation nodes can now be duplicated via standard copy/paste (Ctrl/Cmd+C →
Ctrl/Cmd+V). The pasted node is offset slightly from the original and
immediately selected.

### Screenshot

Updated README screenshot to reflect the current UI (ISE policy set with full
authen/authz chain, policy panel, and node click popover).

## What Shipped in 3.0.2 (patch)

> Released 2026-03-05. Full notes: [`docs/releases/3.0.2.md`](3.0.2.md)

- **Export: black background fix** — transparent PNG composite approach ensures
  clean white backgrounds on PDF/PNG exports regardless of the transparent-bg
  toggle.
- **Export: sharpness improvement** — `EXPORT_MAX_PIXELS` raised to 64 MP;
  typical diagrams now render at pixelRatio 3–4.
- **Export: PDF file size** (community PR #3) — JPEG intermediate reduces
  typical PDF exports from ~70 MB to ~1.8 MB.
- **Export: viewport-independent capture** (community PR #3) — direct
  `.react-flow__viewport` targeting replaces the fragile `fitView()` +
  double-rAF approach.
- **UI: version watermark** — `vX.Y.Z` displayed bottom-left, sourced from
  `package.json`.

## What Shipped in 3.0.1 (patch)

> Released 2026-03-04. Full notes: [`docs/releases/3.0.1.md`](3.0.1.md)

- **Normalizer: numeric RHS display preference** — decision node labels now
  show `displayValue` (e.g. `Wireless-802.11 (19)`) instead of bare numeric IDs
  for RADIUS attributes.
- **Annotation editor: text alignment** — left / center / right toolbar
  buttons added.
- **Annotation editor: text color picker** — inline color applied to selected
  text.
- **Annotation editor: larger font sizes** — 28 pt and 36 pt added to dropdown.
- **Annotation handles cleaned up** — extra source handles removed; target
  handle retained.
- **Handle visibility** — connection handles hidden on all non-annotation nodes
  to reduce visual noise.
- **MIT license** — `LICENSE` file added; `frontend/package.json` updated.
